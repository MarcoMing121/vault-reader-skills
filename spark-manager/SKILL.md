# Spark Manager - 灵感管理器

管理研究灵感的完整生命周期：从种子到结果（或枯萎）。

## 目录结构

```
灵光一现/
├── 种子/      🌱 刚冒出的想法，未探索
├── 发芽/      🌿 开始探索，查阅相关论文
├── 成长/      🌳 形成具体假设，设计实验
├── 结果/      🍎 验证完成，产出结论
└── 枯萎/      🥀 放弃的想法，保留原因
```

## 灵感状态

| 状态 | Emoji | 含义 | 进入条件 | 超时提醒 |
|------|-------|------|----------|----------|
| 种子 | 🌱 | 刚冒出的想法 | 新建灵感 | 7 天未更新 |
| 发芽 | 🌿 | 开始探索 | 链接了概念/论文 | 14 天未更新 |
| 成长 | 🌳 | 形成假设 | 写了假设或实验计划 | 30 天未更新 |
| 结果 | 🍎 | 验证完成 | 有实验结论 | 无 |
| 枯萎 | 🥀 | 放弃 | 手动标记 | 无 |

## Step 1: 创建新灵感

当用户说 "记录灵感"、"有个想法" 时：

### 1.1 询问核心信息

- **标题**：一句话概括
- **核心问题**：想解决什么？
- **来源**：这个想法怎么来的？（读书、讨论、论文...）

### 1.2 生成灵感文件

路径：`{VAULT_PATH}/灵光一现/种子/YYYY-MM-DD-标题.md`

```markdown
---
title: 小动作与自我意识
status: 🌱 种子
created: 2026-03-30
updated: 2026-03-30
source: 读书时的突然想法
trigger: "《具身认知》第二章"
---

## 核心问题

人类的小动作是否能帮助网络更好地认识自己？

## 来源

读《具身认知》时想到的...

## 假设

<!-- 成长阶段填写 -->
- [ ] 假设1：
- [ ] 假设2：

## 关联概念

<!-- 手动添加，格式：[[概念名]] -->

> 💡 **提示**：告诉我"添加概念 [[概念名]]"，我会帮你链接到灵感

## 支撑论文

<!-- 手动添加，格式：[[论文名]] 或 arXiv ID -->

> 💡 **提示**：告诉我"链接论文 [[论文名]]"或"找相关论文"，我会帮你发现关联

## 实验/验证思路

<!-- 成长阶段填写 -->

## 进展记录

- 2026-03-30: 💡 想法诞生

## 下一步

- [ ] 思考这个想法的核心假设是什么
- [ ] 找相关论文或概念
```

### 1.3 告知用户

```
✅ 灵感已记录！

📝 标题: 小动作与自我意识
📍 位置: 灵光一现/种子/2026-03-30-小动作与自我意识.md

💡 下一步可以：
1. 告诉我"添加概念 [[概念名]]"链接相关概念
2. 告诉我"链接论文 [[论文名]]"或提供 arXiv ID
3. 说"帮我找相关论文"让我自动发现（需 concept-weaver）
```

## Step 2: 更新灵感状态

当用户提到已有灵感时，检查是否需要状态升级：

### 2.1 自动检测升级条件

| 当前状态 | 升级条件 | 新状态 |
|----------|----------|--------|
| 种子 | 链接了概念或论文 | 发芽 |
| 发芽 | 写了假设或实验计划 | 成长 |
| 成长 | 标记为"验证完成" | 结果 |
| 任意 | 标记为"放弃" | 枯萎 |

### 2.2 移动文件到新目录

```bash
mv 灵光一现/种子/xxx.md 灵光一现/发芽/xxx.md
```

### 2.3 更新文件内容

- 修改 `status` 字段
- 更新 `updated` 日期
- 添加进展记录

## Step 3: 手动添加关联

### 3.1 添加概念

用户说：`给灵感"小动作与自我意识"添加概念 [[本体感觉]]`

执行：
1. 找到对应灵感文件
2. 在"关联概念"下添加 `- [[本体感觉]]`
3. 更新 `updated` 日期
4. 检查是否需要升级状态（种子→发芽）

### 3.2 添加论文

用户说：`给灵感"小动作与自我意识"链接论文 [[LeWorldModel]]`

或：`链接论文 arXiv:2403.12345`

执行：
1. 找到对应灵感文件
2. 在"支撑论文"下添加链接
3. 更新 `updated` 日期
4. 检查是否需要升级状态

### 3.3 自动发现（可选）

用户说：`帮我找这个灵感的相关论文`

执行：
1. 读取灵感的核心问题和来源
2. 调用 concept-weaver 或搜索论文库
3. 返回候选列表
4. 用户确认后添加

## Step 5: 定期回顾

### 5.1 超时提醒

OpenClaw cron 任务，每周一 9:00 检查：

- 🌱 种子 > 7 天未更新
- 🌿 发芽 > 14 天未更新
- 🌳 成长 > 30 天未更新

提醒格式：

```
⏰ 灵感回顾提醒

以下灵感需要关注：

🌱 种子超时 (2)
  - 小动作与自我意识 (12 天未更新)
    → 建议：添加概念或链接论文，让它发芽

🌿 发芽超时 (1)
  - ...

💡 快速操作：
- "更新灵感"xxx"状态"
- "放弃灵感"xxx"
```

### 5.2 每月统计

每月 1 号 9:00 发送灵感月报：

```
📊 灵感月报 - 2026年3月

总计: 6 个灵感

| 状态 | 数量 | 本月新增 |
|------|------|----------|
| 🌱 种子 | 3 | +2 |
| 🌿 发芽 | 2 | +1 |
| 🌳 成长 | 1 | 0 |
| 🍎 结果 | 0 | 0 |
| 🥀 枯萎 | 0 | 0 |

本月亮点:
- "小动作与自我意识" 从种子成长到发芽

待关注:
- 2 个种子超过 7 天未更新
```

## Step 6: 标记放弃

用户说：`放弃灵感"xxx"，原因：xxx`

执行：
1. 移动到 `枯萎/` 目录
2. 在文件末尾添加：

```markdown
## 枯萎记录

- **放弃时间**: 2026-03-30
- **放弃原因**: 查阅文献后发现已有类似研究，且我的切入点不够新
- **经验教训**: 下次先查文献再深入
```

## Step 0: 读取共享配置

先读取 `../_shared/user_config.py`，如果 `../_shared/user-config.local.json` 存在，再用它覆盖默认值。

显式生成并在后续统一使用这些变量：

- `VAULT_PATH` — Obsidian vault 根目录
- `SPARK_PATH` — 灵感目录（默认 `{VAULT_PATH}/灵光一现`）

## 配置

灵感目录路径从 `../_shared/user_config.py` 读取：

```json
{
  "VAULT_PATH": "<your vault path>",
  "SPARK_PATH": "<vault path>/灵光一现"
}
```

## 注意事项

1. **不要过度自动化** - 用户手动添加概念/论文更精准
2. **保留历史** - 枯萎的灵感不删除，保留学习价值
3. **状态可逆** - 允许从成长退回到发芽（发现假设有问题）
4. **定期提醒但不打扰** - 只在超时时提醒，不频繁骚扰
